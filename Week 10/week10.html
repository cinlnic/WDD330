Week 10 Notes

Validating Forms

Client side form validation
verify all form fields are filled out

Server side form validation
done on the server after form is submitted

Built in form validation - uses validation features
required - specifies whether a form field must be filled in before submitted
minlength and maxlength - the min and max length of data
min and max - specifies min and max values of number inputs
pattern - specifies a regex pattern

When an element matches :valid CSS psuedo-class lets you apply a specific style to a valid element

Validating forms using JavaScript

most browsers support Constraint Validation API

Fetch APi

fetch() - method that lets you fetch resources asynchronously across the network
Takes one argument - the path to the resource you want to fetch
returns a promise that resolves with a response object

response.json() - will return the JSON object

Check for a successful fetch() would include checking the promise was resolved, then checking the respones.ok property has a value of true
