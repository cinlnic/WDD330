Week 1 Notes

Errors, Exceptions, and Warnings

Errors are usually caused by system error, programmer error, or user error.

Exceptions
error that produces a return value that can then be used by the program to deal with the error.

Warning
error in the code that isn't enough to cause the program to crash.

Strict Mode

Produces more exceptions and warnings and prohibits the use of some deprecated features.
requires 'use strict'; to be added to the first line of a JS file.
Can be used on a per-function basis by adding the line inside a function.

Linting Tools
JS Lint, JS Hint, ES Lint: tools that can be used to test the quality of JS Code

Feature Detection

determines browser support for a feature
use an if statement to check whether an object or method exists

Debugging in the Browser

alert() - most basic form of debugging, stops a program from running until OK is clicked and allows variables to be checked

console object - logs information debugging

debugging tools - allows breakpoints to be set

Error Objects

can be created by the host enviroment when an exception occures, or created in the code using a constructor function: const error = new Error(); takes a parameter that's used as error message
it is possible to throw your own exceptions using the throw statement.

Exception Handling

When exceptions occur the program terminates with an error message. This does not look good in production. Catching the error will allow the error to be hidden from users and possibly let the program keep running.

try, catch, finally

Tests

test can be a simple function that tests a piece of code runs as it should

Test-driven development - process of writing tests before any actual code.
