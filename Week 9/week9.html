Week 9 Notes

Window Object

The Browser Object Model

properties and methods that contain information about the browser and computer screen
available through the window object

Going Global
global variables can be accessed in all parts of the program
global variables are properties of the global object

Dialog
alert()
confirm()
prompt()
these stop the execution of the program until the user clicks ok or cancel

Browser Information

window.navigator.userAgent - returns info about the browser and operating system, can be modified so may not be accurate

Location Properties
window.location - contains info abou the URL of the current page
window.location.href - returns the URL as a string
window.location.protocol - returns a string describing the protocol used (http, https, pop2, ftp, etc.)
window.location.host - returns a string describing the comain of the current URL and the port number
window.location.hostname - returns a string describing the domain of the current URL
window.location.port - returns the port number if it is explicitly stated in the URL
window.location.pathname - returns the path that follows the domain
window.location.search - returns a string that starts with a ? followed by the query string
window.location.hash - returns a string starting with a # followed by the fragment identifier
window.location.origin - shows protocol and domain where the current page originated from

Location Methods
reload() - force a reload of the current page
assign() - load anther resource from a URL
replace() - same as assign, but the current page will not be stored in the session history so user can not navigate back to it
toString() - returns a string containing the whole URL

The Browser History

window.history can be used to access info about any previously visited page in the current browser session
window.hisotry.length - shows how many pages have been visited before arriving at the current page
window.history.go() - can be used to go to a specific page, where 0 is the current page can also use window.hisotry.forward() and window.history.back()

Controlling Windows

window.open() - opens a new window, takes the URL of the page to be opened as first parameter, window title as second, list of attributes as the third
close() - used to close a window
window.moveTo() - can move a window using X,Y coordinates
window.resizeTo() - change height and width of a window

Screen Information

window.screen object contains info about the screen the browser is displayed on

The Document Object

document.write() - writes a string to the page, if a page is loaded it will completely replace the current document

Cookies
small files saved locally on user's computer
creating cookies - document.cookie = 'name=Superman'
cookie property acts like a special type of string, assinging another cookie won't overwrite the entire property, just append it to the end of the string
Changing values - a cookies value can be changed by reassinging it
Reading cookies - enter document.cookie:
Cookie expiry dates - deleted when a browser session is finished, can be made persistent by adding 'expires=date' to the end of the cookie when set or using max-age
Path and domain of cookies
path=/ - any page in the root directory can read the cookie
domain=domainName

Secure Cookies
add string secure to the end will ensure it's only transmitted over a secure HTTPS network

Deleting cookies
to remove a cookie set it to expire at time in the past

Timing Function

setTimeout(), clearTimeout()

setInterval() - repeatedly invoke callback function after every given number of miliseconds

HTML5 API

The data- Attribute

a way of embedding data in a web page using custom attributes that are ignored by the browser
start with data-
contain only lowercase letters, numbers, hyphens, dots, colons or underscores
include an option string value
example: data-powers = 'flight superSpeed'
each element has a dataset property used to access the data- attributes it contains
const powers = superman.dataset.powers; returns 'flight superSpeed'

HTML5 Web Storage

Info is not shared with the server on every request
Info is available in multiple windows of the browser if the domain is the same
Storage capacity limit is much larger than cookie limit
Data stored does not expire

localstorage.setItem('name', 'Walter White')

localStorage.getItem('name');

localstorage.removeItem('name'); or delete localStorage.name;

localStorage.clear(); - removes everything in localStorage
