This in JavaScript

"This" is a keyword whose value changes depending on how a function gets called.

Object

self-contained set of related values and functions. Properties = value. If the value is a function it is known as a method.

Object Literal
created directly by wrapping all its properties and methods in {}
create an object literal: const spiderman = {};

Accessing Properties
dot notation: superman.name
bracket notation: superman['name']

Calling Methods
dot notation: superman.fly()
bracket notation: superman['fly']()

Checking if properties exist
in operator: 'city' in superman;
checking if it doesn't return undefined: superman.city !== undefined;
hasOwnProperty(): superman.hasOwnProperty('city');

Finding all properties of an object
fo in loop: for(const key in superman) { console.log(key + ": " + superman[key]);
Object.keys() will return an array of all the keys of an object
Object.values() will return an array of all the object's value
Object.entries() returns an array of key-value pairs

Adding properties: done by assigning a value to the new property
superman.city = 'Metropolis';

Changing properties: use assingment
superman['real name'] = 'Kal-El';

Removing properties: use the delete operator
delete superman.fly

Nested Objects: objects can contain other objects
values can be accessed by referencing each property in order using dot or bracket notation
jla.wonderWoman.realName or jla.['flash']['realName']

Namespacing: create an object literal, then put all values as properties and functions as methods.
const myMaths = { square(x){return x*x} }; called by using the namespace: myMaths.square(3)

Built in objects: JSON methods, Math object, Date object, RegExp object

The Document Object Model

Getting Elements
DOM level 0: document.body, .images, .links, .anchors, .forms
array like objects, but not actually arrays
have a length property, but no other array methods
can be made into a node list using Array.from(document.images) or [...document.images]

document.getElementby: Id(), TagName(), ClassName(),

document.querySelector(): finds the first element in the document matching the selector provided
document.querySelectorAll(): finds all elements matching selector provided
can also select psuedo elements
can be called on any element not just document

Getting and Setting Attributes
getAttribute(): returns the value of the attribute
setAttribute(): changes the value of an element's attribute, also used to add an attribute to an element

Classes of an Element
className property: allows the class to be set directly, and also to find the value of the class attribute
wonderWoman.className; returns "villain" or wonder.Woman.className = 'hero' - changes the class from villain to hero
classList property: list of all classes an element has, has methods to modify the class of an element
add: used to add a class without overwriting other classes that already exist
remove: removes a specific class from an element
toggle: will add a class if an element doesn't already have it and remove it if it does
contains: check to see if an element has a class

Dynamic Markup
createElement(): takes a tag name as a parameter and returns that element - const flash = document.createElement('li');
createTextNode(): parameter is a string containing the text that goes in the node - const flashText = document.createTextNode('Flash');
appendChild(): add another node as a child node - flash.appendChild(flashText)
textContent: makes adding content to an element without appending it - flash.textContent = 'Flash';
appendChild() adds new element at the bottom of a list
insertBefore(): will place the new element before another element - heroes.insertBefore(aquaman, wonderWoman);
removeChild(): removes elements from a page
replaceChild(): used to replace on node with another
innerHTML: returns all raw HTML and writable

Events
Event Listeners: document.body.addEventListener("click", doSomething) - this will listen for a click anywhere on the page, then call the doSomething() function
addEventListener(): first parameter is the type of event, second is the callback function
Inline event handlers:
onclick: inline attributes, best avoided
